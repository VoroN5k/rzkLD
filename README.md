# 🎬 HLS Downloader

Маленька програма, яка вміє зберігати відео, що транслюється потоково (HLS / `.m3u8`), у звичайний `.mp4` файл на твоєму комп'ютері.

## 📥 Встановлення

Перейди на сторінку **[Releases](../../releases)** і завантаж версію під свою систему.

### Windows

- Файл `.msi` — повноцінний інсталятор (з'явиться в меню Пуск, можна видалити через "Програми і компоненти")
- Файл `.exe` — portable-варіант, просто запускається без встановлення

Якщо Windows покаже попередження "Невідомий видавець" — це нормально для невеликих застосунків без платного сертифіката. Тисни **"Виконати все одно" / "More info → Run anyway"**.

### Linux

- Файл `.deb` (Ubuntu/Debian):
  ```bash
  sudo dpkg -i hls-downloader_*.deb
  ```
- Файл `.AppImage` (працює майже на будь-якому дистрибутиві, без встановлення):
  ```bash
  chmod +x HLS_Downloader_*.AppImage
  ./HLS_Downloader_*.AppImage
  ```

## ⚙️ Обов'язкові залежності

Програма сама лише качає й клікає — для конвертації відео і для авто-пошуку плейлиста потрібні дві зовнішні програми, встановлені окремо.

### 1. ffmpeg — для конвертації в mp4

**Windows:**
```
winget install ffmpeg
```
Якщо `winget` не знайдено — завантаж вручну з [ffmpeg.org](https://ffmpeg.org/download.html) і додай шлях до `ffmpeg.exe` у системну змінну PATH.

**Linux:**
```bash
sudo apt install ffmpeg
```

**Перевірка (обидві системи):**
```
ffmpeg -version
```
Якщо команда показує версію — все ок.

### 2. Справжній Chrome/Chromium — для функції "Знайти m3u8"

Потрібен тільки для авто-пошуку плейлиста через сторінку плеєра (розділ нижче). Просте завантаження по готовому `.m3u8`-посиланню без цього теж працює.

**Windows:** звичайний встановлений Chrome чи Edge — зазвичай вже є в системі.

**Linux:** тут є нюанс. Пакет `chromium-browser` із `apt` на сучасних Ubuntu (22.04+) — це **порожня заглушка**, яка вимагає `snap`, а snap не завжди працює коректно (особливо в контейнерах/мінімальних системах). Надійніше поставити офіційний `.deb` від Google:
```bash
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install ./google-chrome-stable_current_amd64.deb
```
Якщо в тебе вже встановлений Chromium і він реально запускається (не просить snap) — теж підійде, додатково нічого робити не треба.

**Якщо Chrome лежить у нестандартному місці** — встанови змінну середовища `CHROME` з повним шляхом до виконуваного файлу перед запуском програми.

## ▶️ Як користуватись: пряме завантаження

1. Відкрий програму
2. Встав посилання на `.m3u8`-плейлист у поле **"URL плейлиста"**
3. (опціонально) зміни назву файлу в полі **"Зберегти як"**
4. (опціонально) тисни 📁 і вибери папку, куди зберегти результат — якщо не вибрати, файл з'явиться поряд із самою програмою
5. Тисни **"Завантажити"**
6. Зачекай, поки прогрес-бар дійде до 100% і програма сконвертує файл у mp4

## 🤖 Автопошук плейлиста (для власного плеєра)

Якщо в тебе свій плеєр і ти не хочеш шукати `.m3u8` вручну через DevTools браузера — є автоматичний режим.

1. Встав URL сторінки плеєра в поле **"URL сторінки плеєра"**
2. Якщо кнопка play знаходиться нижче видимої частини сторінки — встанови значення в полі **"Скрол вниз"** (за замовчуванням `400`)
3. Натисни **"📷 Зробити скріншот"** — застосунок відкриє сторінку в headless-браузері, проскролить і збереже кадр `debug_screenshot.png` поряд із програмою
4. Відкрий цей файл (наприклад у Paint на Windows чи GIMP на Linux) і наведи курсор на кнопку play — програма перегляду покаже координати X/Y
5. Впиши ці X/Y у відповідні поля (за замовчуванням стоять `484`/`293` — просто приклад, для твого плеєра потрібні саме твої значення)
6. Тисни **"🔍 Знайти m3u8"**

**Якщо кнопка лежить у DOM головної сторінки** (не всередині `<iframe>`) — можна замість координат указати CSS-селектор кнопки в полі вище, тоді координати не потрібні.

**Якщо результат — "Жодного мережевого запиту взагалі не зафіксовано"** — клік не потрапив на елемент. Перевір координати ще раз на свіжому скріншоті (розмір вікна і скрол впливають на позицію).

**Якщо результат показує список URL без `.m3u8`** — клік спрацював, але потрібного запиту серед видимих немає. Можливо, потрібно зачекати довше, або потік іде в іншому форматі.

## 🔍 Альтернатива: знайти посилання вручну через DevTools

1. Відкрий відео в браузері
2. `F12` → вкладка **Network** → фільтр `m3u8`
3. Запусти відтворення відео
4. У списку з'явиться запит, що повертає текст, який починається з `#EXTM3U`

## ⚠️ Важливо

Програма — звичайний технічний інструмент для роботи з HLS-потоками (як `ffmpeg` чи `yt-dlp`). Використовуй її тільки для контенту, на перегляд/збереження якого маєш право: власні відео, відкриті/публічні джерела, легальні сервіси, що дозволяють офлайн-перегляд.

## 🐛 Якщо щось не працює

- **"ffmpeg не знайдено в PATH"** — встанови ffmpeg (див. вище) і перезапусти програму
- **Завантаження зависає на 0%** — перевір, що посилання справді веде на `.m3u8`-файл (відкрий його прямо в браузері, має показати текст, що починається з `#EXTM3U`)
- **Файл завантажився, але не відкривається** — спробуй відкрити через VLC; якщо й там не йде, можливо потік зашифрований (AES-128) — ця версія програми такого не підтримує
- **На Linux: помилка про відсутній Chrome/Chromium** — дивись секцію залежностей вище, проблема майже завжди в snap-заглушці замість справжнього браузера
